# Prior JavaScript Experience — Some Exposure, Gaps Unknown

User self-reports "some JS experience" — knows basic JS but cannot yet build things confidently on their own. Goal is a developer job, meaning they need interview-level fluency, not just working familiarity.

**Evidence:** Self-reported in session intake.

**Implications:** Do not re-teach syntax from scratch with a beginner's pace. Start at a level that would be revision for a beginner but exposes the *why* behind common patterns — this is exactly where interview gaps live. Prioritize depth over breadth in early lessons to correct likely misconceptions (hoisting, type coercion, scope, closures).
