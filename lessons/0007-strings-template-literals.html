1. String Basics

Strings in JavaScript are immutable sequences of characters. You cannot change an individual character in a string — every "transformation" creates a brand new string and leaves the original untouched.

const str = "hello";

// length property — number of characters
console.log(str.length);    // 5

// Accessing characters by index (0-based)
console.log(str[0]);         // "h"
console.log(str[10]);        // undefined (out of bounds)

// charAt — older alternative
console.log(str.charAt(0));  // "h"
console.log(str.charAt(10)); // "" (empty string — different from undefined)

// Strings are compared lexicographically (dictionary order)
console.log("apple" < "banana");  // true
console.log("Z" < "a");          // true — uppercase letters have lower char codes
Immutability in practice str[0] = "H" silently does nothing in non-strict mode — it won't throw an error, but the string is unchanged. This surprises many developers.

const sentence = "The quick brown fox jumps over the lazy dog";

console.log(sentence.indexOf("fox"));        // 16
console.log(sentence.indexOf("cat"));        // -1 (not found)
console.log(sentence.lastIndexOf("the"));    // 31
console.log(sentence.includes("fox"));       // true
console.log(sentence.startsWith("The"));    // true
console.log(sentence.endsWith("dog"));      // true

// All methods are case-sensitive
console.log(sentence.includes("Fox"));       // false

3. Extracting Substrings

There are three ways to extract a portion of a string. slice is the one you should use and memorize.

const str = "Hello, World!";
//            0123456789...

// slice(start, end) — end is EXCLUDED
console.log(str.slice(0, 5));    // "Hello"
console.log(str.slice(7));       // "World!" (no end = go to end)
console.log(str.slice(-6));      // "World!" (negative = count from end)
console.log(str.slice(-6, -1)); // "World"

// substring(start, end) — same idea but no negative indices
console.log(str.substring(0, 5)); // "Hello"
console.log(str.substring(-6));  // "Hello, World!" — negative treated as 0

// at(index) — modern, supports negative indexing cleanly
console.log(str.at(0));          // "H"
console.log(str.at(-1));         // "!" (last character)
Recommendation Use slice() for everything. It supports negative indices (count from end), is consistent, and is not deprecated. Avoid substr() — it's been removed from the standard.

4. Transforming Strings

Remember: every method returns a new string. The original is unchanged.

const str = "  Hello, World!  ";

// Case
console.log(str.toUpperCase());        // "  HELLO, WORLD!  "
console.log(str.toLowerCase());        // "  hello, world!  "

// Trimming whitespace
console.log(str.trim());               // "Hello, World!"
console.log(str.trimStart());          // "Hello, World!  "
console.log(str.trimEnd());            // "  Hello, World!"

// Padding — great for formatting numbers, IDs, time displays
console.log("7".padStart(3, "0"));    // "007"
console.log("hi".padEnd(5, "."));    // "hi..."

// Repeat
console.log("ha".repeat(3));           // "hahaha"

// Replace
const msg = "I love cats. Cats are great.";
console.log(msg.replace("cats", "dogs"));    // "I love dogs. Cats are great." — first only
console.log(msg.replaceAll("cats", "dogs")); // still only "I love dogs..." — case sensitive!

// Case-insensitive replace requires a regex
console.log(msg.replace(/cats/gi, "dogs"));  // "I love dogs. dogs are great."

5. Splitting & Joining

split converts a string into an array; array's join does the reverse. Together they enable many string manipulation patterns.

// split(separator) — string → array
const csv = "alice,bob,carol";
console.log(csv.split(","));         // ["alice", "bob", "carol"]
console.log(csv.split(",", 2));      // ["alice", "bob"] — limit parameter
console.log("hello".split(""));     // ["h","e","l","l","o"] — split every char

// join(separator) — array → string
const words = ["one", "two", "three"];
console.log(words.join(" - "));     // "one - two - three"
console.log(words.join(""));        // "onetwothree"

// Classic pattern: reverse a string
const reversed = "hello".split("").reverse().join("");
console.log(reversed);              // "olleh"
Unicode caution The split/reverse/join trick breaks for emoji and multi-byte characters because they consist of multiple UTF-16 code units. For production use, prefer the spread operator: [...str].reverse().join(""), which handles Unicode better.

6. Template Literals

Template literals (backtick strings) are one of the most important ES6 features. They replace string concatenation and enable multi-line strings cleanly.

const name = "Karan";
const score = 95;

// Old way — concatenation (ugly, error-prone)
const msg1 = "Hello, " + name + "! Your score is " + score + ".";

// Template literal — interpolation with ${}
const msg2 = `Hello, ${name}! Your score is ${score}.`;

// Any expression works inside ${}
const msg3 = `${name.toUpperCase()} scored ${score >= 90 ? "A" : "B"}`;
console.log(msg3); // "KARAN scored A"

// Multi-line strings — no \n needed
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>Score: ${score}</p>
  </div>
`;

Tagged Template Literals

A tag is a function placed in front of a template literal. The function receives the string parts and interpolated values separately, giving you total control over how the string is assembled. This powers libraries like styled-components and safe SQL query builders.

// Tag function signature: (strings, ...values)
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const val = values[i] !== undefined
      ? `<strong>${values[i]}</strong>`
      : "";
    return result + str + val;
  }, "");
}

const user = "Karan", score = 95;
console.log(highlight`Hello ${user}, you scored ${score}!`);
// "Hello <strong>Karan</strong>, you scored <strong>95</strong>!"

7. String Gotchas

These are the common traps that catch even experienced developers.

Type Conversions

// Number → String
console.log(String(42));         // "42" — explicit, preferred
console.log((42).toString());    // "42"
console.log(42 + "");           // "42" — implicit, works but less readable

// String → Number
console.log(Number("42"));       // 42 — explicit, preferred
console.log(parseInt("42px"));   // 42 — stops at first non-numeric char
console.log(parseFloat("3.14em")); // 3.14
console.log(+"42");             // 42 — unary plus, works but cryptic
console.log(Number(""));         // 0 — watch out!
console.log(Number("abc"));      // NaN
NaN is never equal to itself NaN === NaN is false — this is the one value in JavaScript that is not equal to itself. Always use Number.isNaN(value) to check for NaN. Never use value === NaN. 
// Checking for NaN
const result = Number("abc");   // NaN

result === NaN;                  // false — always false, useless
isNaN(result);                   // true — but also returns true for isNaN("abc") (coerces first)
Number.isNaN(result);            // true — correct, no coercion
Number.isNaN("abc");             // false — "abc" is not NaN, it's a string
