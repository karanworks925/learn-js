Destructuring, Spread & Rest

ES6's most expressive syntax — write half the code, say twice as much.

Primary Source for this lesson Read this after the lesson: javascript.info — Destructuring Assignment. The section on nested destructuring and function parameter destructuring is especially worth re-reading.

1. Array Destructuring

Destructuring lets you unpack values from arrays or properties from objects into distinct variables — in a single, readable statement. Instead of writing arr[0], arr[1], you declare exactly what you need.

// Basic array destructuring
const [a, b, c] = [1, 2, 3];
console.log(a, b, c); // 1 2 3

// Skipping elements with a hole (empty comma)
const [first, , third] = [10, 20, 30];
console.log(first, third); // 10 30

// Default values — used when the value is undefined
const [x = 1, y = 2] = [10];
console.log(x, y); // 10 2  ← y uses its default

// Rest in array destructuring
const [head, ...tail] = [1, 2, 3, 4];
console.log(head); // 1
console.log(tail); // [2, 3, 4]

// Swap two variables without a temp variable
let p = 5, q = 10;
[p, q] = [q, p];
console.log(p, q); // 10 5
Variable swap The swap trick [a, b] = [b, a] is a one-liner that used to require a temporary variable. It's idiomatic modern JS — you'll see it in real codebases.

2. Object Destructuring

Object destructuring extracts properties by name, not by position. The variable names must match the property keys — unless you rename them.

const user = { name: "Karan", age: 24, city: "Delhi" };

// Basic — variable names must match property keys
const { name, age } = user;
console.log(name, age); // Karan 24

// Renaming: { key: newVariableName }
const { name: fullName, age: years } = user;
console.log(fullName, years); // Karan 24

// Default values — used when the property is undefined
const { role = "guest" } = user;
console.log(role); // "guest" — user has no role property

// Nested destructuring
const profile = {
  username: "karan99",
  address: { city: "Delhi", country: "India" },
};
const { address: { city, country } } = profile;
console.log(city, country); // Delhi India

// Rest in object — collect remaining properties
const { name: n, ...rest } = user;
console.log(n);    // Karan
console.log(rest); // { age: 24, city: "Delhi" }
Renaming syntax is backwards to what you might expect const { name: fullName } = user means "get the property called name, and put it in a variable called fullName." The colon reads as "rename to", not "the value is".

3. Function Parameter Destructuring

You can destructure directly in the function parameter list. This is extremely common when functions receive an options/config object — it documents exactly what properties the function needs.

// Without destructuring — must write user.name, user.age everywhere
function greetOld(user) {
  return `Hi ${user.name}, you are ${user.age}`;
}

// With destructuring — clean, self-documenting
function greet({ name, age = 0 }) {
  return `Hi ${name}, you are ${age}`;
}

greet({ name: "Karan", age: 24 }); // "Hi Karan, you are 24"
greet({ name: "Alice" });         // "Hi Alice, you are 0" — default kicks in

// Array parameter destructuring
function sum([a, b]) {
  return a + b;
}
sum([3, 7]); // 10

// Real-world: event handler destructuring
document.addEventListener("click", ({ target, clientX, clientY }) => {
  console.log(target, clientX, clientY);
});

4. Spread Operator with Arrays

The spread operator ... expands an iterable (like an array) into individual elements. In array contexts it lets you copy, merge, and insert arrays without mutation.

const arr = [1, 2, 3];

// Shallow copy
const copy = [...arr];
copy.push(4);
console.log(arr);  // [1, 2, 3] — original unchanged
console.log(copy); // [1, 2, 3, 4]

// Merge two arrays
const a = [1, 2];
const b = [3, 4];
const merged = [...a, ...b]; // [1, 2, 3, 4]

// Insert at a specific position
const withZero = [0, ...arr, 100]; // [0, 1, 2, 3, 100]

// Spread as function arguments
const nums = [3, 1, 4, 1, 5];
Math.max(...nums); // 5 — same as Math.max(3, 1, 4, 1, 5)

// Convert a string or NodeList to an array
const chars = [..."hello"]; // ["h","e","l","l","o"]

5. Spread Operator with Objects

Spread also works on objects (ES2018+). It copies own enumerable properties — but only one level deep (shallow copy).

const defaults = { theme: "light", lang: "en", fontSize: 14 };
const userSettings = { theme: "dark", fontSize: 16 };

// Shallow copy of an object
const clone = { ...defaults };

// Merge — later properties win on conflict
const merged = { ...defaults, ...userSettings };
// { theme: "dark", lang: "en", fontSize: 16 }
// theme and fontSize come from userSettings; lang from defaults

// The "defaults override" pattern used in every real app
function createConfig(overrides = {}) {
  return { ...defaults, ...overrides };
}
createConfig({ theme: "dark" });
// { theme: "dark", lang: "en", fontSize: 14 }

// Update a single property without mutating the original (React pattern)
const user = { name: "Karan", age: 24 };
const updatedUser = { ...user, age: 25 };
// { name: "Karan", age: 25 } — user is untouched
Shallow copy warning Both array and object spread are shallow. If a property holds a nested object, only the reference is copied — not the nested object itself. Mutating nested data will still affect the original.

6. Rest Parameters

Rest collects the remaining arguments into a real array. It looks like spread (same ... syntax) but it works in the opposite direction: instead of expanding, it gathers.

// Rest must be the LAST parameter
function sum(...numbers) {
  return numbers.reduce((acc, n) => acc + n, 0);
}

sum(1, 2);          // 3
sum(1, 2, 3, 4);    // 10
sum(10);            // 10

// Mix named params with rest
function log(level, ...messages) {
  console.log(`[${level}]`, ...messages);
}
log("ERROR", "something", "went wrong");
// [ERROR] something went wrong

Rest vs the arguments object

FeatureRest parametersarguments object
Is a real ArrayYes — all array methods workNo — array-like, not Array
Works in arrow functionsYesNo — arrow functions have no arguments
Partial collectionYes — collects only the restNo — always all arguments
arguments is legacy Avoid arguments in new code. Use rest parameters instead. Arrow functions don't even have an arguments object — using it there will reach into the enclosing non-arrow function, which is always surprising.

7. Common Real-World Patterns

// 1. Extract from an API response
const response = {
  status: 200,
  data: { userId: 42, name: "Karan", token: "abc123" },
};
const { data: { userId, name } } = response;
// userId = 42, name = "Karan" — token is ignored

// 2. Function with optional config + defaults
function fetchUser({ id, cache = true, timeout = 5000 } = {}) {
  console.log(`fetching ${id}, cache: ${cache}, timeout: ${timeout}`);
}
fetchUser({ id: 1 });              // uses cache=true, timeout=5000
fetchUser({ id: 2, cache: false }); // overrides cache
fetchUser();                       // = {} prevents crash if called with nothing

// 3. Clone and update state (immutable update pattern)
const state = { count: 0, user: "Karan", loading: false };
const nextState = { ...state, count: state.count + 1, loading: true };
// { count: 1, user: "Karan", loading: true }
