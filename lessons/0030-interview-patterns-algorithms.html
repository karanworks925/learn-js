1. How to Approach a Coding Interview

The code is only half of what you're being evaluated on. Interviewers also assess your communication, your ability to handle ambiguity, and your engineering judgment. Never write code in silence.

  1. Clarify the problem. Ask questions before touching the keyboard. What are the constraints? What's the expected input/output? Are there edge cases? What about empty input, duplicates, very large inputs?
  2. Work through examples aloud. Pick a small concrete example and trace through it manually. This surfaces misunderstandings before you write code.
  3. State your approach before coding. "I'm going to use a hash map to get O(n) time. The trade-off is O(n) space." The interviewer may guide you if you're heading somewhere wrong.
  4. Code the solution, narrating as you go. "I'm creating a set to track seen values... here I'm iterating... this condition handles the edge case where..."
  5. Test with edge cases. After your solution works on the happy path, manually trace through: empty input, single element, negative numbers, duplicates.
  6. Analyze time and space complexity. State Big-O proactively. If you can optimize, say so: "This is O(n²) — I could improve it to O(n) with a hash map if needed."
The #1 Interview Mistake Jumping straight to code without clarifying the problem. Candidates who start coding immediately frequently solve the wrong problem, or solve a simpler version of the real problem. Ask questions. It signals seniority, not uncertainty.

2. Big-O in JavaScript Context

Big-O is the language of complexity. You need to be able to state it quickly and correctly for any code you write.

ComplexityNameJavaScript Examples
O(1)ConstantObject property access obj.key, array index arr[i], Map.get(), Set.has()
O(log n)LogarithmicBinary search on sorted array

The Array.includes() Trap Using arr.includes(val) inside a loop is O(n²) — it's a linear scan inside a linear loop. Replace the array with a Set or Map and the lookup drops to O(1), making the whole thing O(n). This swap comes up in almost every interview.

3. The Hash Map Pattern — O(1) Lookups

The most important interview pattern. When you find yourself doing a nested loop to find pairs or check for previous values, stop — you almost certainly want a Map.

Classic Problem: Two Sum

Given an array of integers and a target, find the indices of two numbers that add up to the target.

// NAIVE: O(n²) — check every pair
function twoSumNaive(nums, target) {
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {
      if (nums[i] + nums[j] === target) {
        return [i, j];
      }
    }
  }
}

// OPTIMAL: O(n) — use a Map to track seen numbers
function twoSum(nums, target) {
  const seen = new Map(); // value → index

  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];

    if (seen.has(complement)) {
      return [seen.get(complement), i]; // found it!
    }

    seen.set(nums[i], i); // remember this number and its index
  }
}

// Example:
twoSum([2, 7, 11, 15], 9); // [0, 1] — 2 + 7 = 9
twoSum([3, 2, 4], 6);        // [1, 2] — 2 + 4 = 6
The Pattern For each element, ask: "have I seen the number I need to pair with this?" Store what you've seen in a Map. One pass, O(n) time, O(n) space. This pattern solves dozens of interview problems.

4. Two Pointers Pattern

Use two pointers when you need to compare or operate on pairs of elements — especially in sorted arrays or strings. Often converts an O(n²) solution to O(n).

// Check if a string is a palindrome (reads same forwards and backwards)
function isPalindrome(s) {
  // Normalize: lowercase, remove non-alphanumeric
  s = s.toLowerCase().replace(/[^a-z0-9]/g, "");

  let left = 0;
  let right = s.length - 1;

  while (left < right) {
    if (s[left] !== s[right]) return false;
    left++;
    right--;
  }
  return true;
}

isPalindrome("A man, a plan, a canal: Panama"); // true
isPalindrome("race a car");                    // false
Two Pointer Variants Pointers starting at both ends (palindrome check, container with most water). Both pointers starting at the left, moving at different speeds (remove duplicates, fast/slow pointer for cycle detection). The key insight: you're avoiding the need to check every possible pair.

5. Sliding Window Pattern

The sliding window maintains a contiguous subarray (or substring) and slides it across the input. It converts many O(n²) brute-force solutions into O(n).

// Max sum subarray of exactly size k
function maxSumSubarray(nums, k) {
  if (nums.length < k) return null;

  // Build the initial window
  let windowSum = 0;
  for (let i = 0; i < k; i++) windowSum += nums[i];

  let maxSum = windowSum;

  // Slide the window: add next element, remove first element
  for (let i = k; i < nums.length; i++) {
    windowSum += nums[i] - nums[i - k]; // slide right
    maxSum = Math.max(maxSum, windowSum);
  }

  return maxSum;
}

maxSumSubarray([2, 1, 5, 1, 3, 2], 3); // 9 — [5, 1, 3]
maxSumSubarray([2, 3, 4, 1, 5], 2);       // 7 — [3, 4]

6. Classic JS-Specific Interview Questions

These implementations come up in JavaScript interviews specifically — know them by heart. If asked, write them from scratch without hesitation.

Debounce — Implement from Memory

function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
// Time: O(1) per call. Delays execution until N ms of silence.

Deep Clone an Object

// Modern approach — handles most cases (not functions, DOM nodes, etc.):
const clone = structuredClone(obj);

// JSON approach — simple but lossy (loses undefined, functions, Dates become strings):
const clone = JSON.parse(JSON.stringify(obj));

// Manual recursive approach — demonstrates understanding:
function deepClone(obj) {
  if (obj === null || typeof obj !== "object") return obj;
  if (Array.isArray(obj)) return obj.map(deepClone);

  const cloned = {};
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      cloned[key] = deepClone(obj[key]);
    }
  }
  return cloned;
}

Flatten a Nested Array

// Built-in — know this:
[1, [2, [3, [4]]]].flat(Infinity); // [1, 2, 3, 4]

// Recursive implementation (to show understanding):
function flatten(arr) {
  return arr.reduce((acc, val) =>
    Array.isArray(val) ? acc.concat(flatten(val)) : [...acc, val],
  []);
}

flatten([1, [2, [3, [4]]]]); // [1, 2, 3, 4]

Implement Array.prototype.map from Scratch

Array.prototype.myMap = function(fn) {
  const result = [];
  for (let i = 0; i < this.length; i++) {
    if (i in this) { // skip holes in sparse arrays
      result.push(fn(this[i], i, this));
    }
  }
  return result;
};

// Verify it works:
[1, 2, 3].myMap(x => x * 2); // [2, 4, 6]
[1, 2, 3].myMap((x, i) => `${i}:${x}`); // ["0:1", "1:2", "2:3"]

Deep Equality Check

function deepEqual(a, b) {
  // Same reference or same primitive
  if (a === b) return true;

  // Either is null or not an object
  if (a === null || b === null) return false;
  if (typeof a !== "object" || typeof b !== "object") return false;

  const keysA = Object.keys(a);
  const keysB = Object.keys(b);

  // Different number of keys
  if (keysA.length !== keysB.length) return false;

  // Recursively compare all keys
  return keysA.every(key => deepEqual(a[key], b[key]));
}

deepEqual({ a: 1, b: { c: 2 } }, { a: 1, b: { c: 2 } }); // true
deepEqual({ a: 1 }, { a: 2 });                           // false

7. What Separates Junior from Senior in Interviews

The difference is rarely about knowing more algorithms. It's about the approach, communication, and depth of thinking.
