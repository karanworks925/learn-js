Arrays: Basics & Core Methods

Arrays are the most-used data structure in JavaScript — know every method cold.

Primary Source for this lesson Bookmark this page — it is the definitive reference: MDN — Array. Read the method list on the left sidebar. Every method covered in this lesson has an MDN page worth reading.

1. Creating Arrays

Arrays in JavaScript are ordered, zero-indexed collections. Under the hood, they are objects — which has important implications.

// Array literal — always prefer this
const empty = [];
const fruits = ["apple", "banana", "cherry"];
const mixed  = [1, "two", true, null]; // JS allows mixed types (avoid in practice)

// new Array() — avoid; confusing behavior
new Array(3)      // [ <3 empty slots> ] — NOT [3]
new Array(1, 2, 3) // [1, 2, 3] — different behavior with 1 vs multiple args

// Array.from() — create from iterables or with a mapping function
Array.from("hello")         // ["h","e","l","l","o"]
Array.from({ length: 5 }, (_, i) => i) // [0, 1, 2, 3, 4]
Array.from(new Set([1, 2, 2, 3])) // [1, 2, 3] — deduplicate with Set

// Array.of() — creates from arguments (avoids the new Array(3) confusion)
Array.of(3)        // [3] — just the number 3
Array.of(1, 2, 3)  // [1, 2, 3]

// Arrays are objects — use Array.isArray() to check
typeof []           // "object" — not helpful
Array.isArray([])   // true  — correct way
Array.isArray({})   // false

2. Accessing and Modifying Elements

const arr = ["a", "b", "c", "d"];

// Zero-based indexing
arr[0]              // "a" — first element
arr[arr.length - 1] // "d" — last element (classic way)
arr.at(-1)          // "d" — last element (modern ES2022 way)
arr.at(-2)          // "c" — second to last
arr[99]             // undefined — no error for out-of-bounds access

// Direct assignment — arrays are dynamic
arr[1] = "B";        // ["a", "B", "c", "d"]
arr[10] = "z";       // creates sparse array with empty slots 4-9 — avoid this

// .length is always one more than the highest index
console.log(arr.length); // 11 (after the arr[10] assignment above)
Use .at() for Last Elements arr.at(-1) is cleaner than arr[arr.length - 1]. It also works on strings. Available in all modern browsers and Node 16.6+.

3. Adding and Removing Elements (Mutating Methods)

These methods change the original array in place. Know what each one returns — that trips people up in interviews.

MethodWhat it doesReturnsMutates?
push(...items)Add to endNew lengthYes
pop()Remove from endRemoved elementYes
unshift(...items)Add to startNew lengthYes
shift()Remove from startRemoved elementYes
splice(start, deleteCount, ...items)Insert/remove anywhereArray of removed itemsYes
 
const arr = [1, 2, 3];

// push — returns NEW LENGTH (not the array!)
const newLen = arr.push(4);  // arr = [1,2,3,4], newLen = 4

// pop — returns the REMOVED ELEMENT
const last = arr.pop();      // arr = [1,2,3], last = 4

// unshift — adds to the START, returns new length
arr.unshift(0);              // arr = [0,1,2,3]

// shift — removes from START, returns removed element
const first = arr.shift();   // arr = [1,2,3], first = 0

// splice — the Swiss Army knife
const items = ["a", "b", "c", "d"];

// Remove 2 elements starting at index 1
const removed = items.splice(1, 2); // removed = ["b","c"], items = ["a","d"]

// Insert without removing: deleteCount = 0
items.splice(1, 0, "x", "y");    // items = ["a","x","y","d"]

// Replace elements
items.splice(2, 1, "Z");          // replace "y" with "Z"

4. Non-Mutating Methods

These methods return a new array and leave the original untouched. Prefer these in functional-style code.

const original = [1, 2, 3, 4, 5];

// slice(start, end) — copy a portion (end index is EXCLUDED)
original.slice(1, 3)   // [2, 3] — indices 1 and 2 only
original.slice(2)      // [3, 4, 5] — from index 2 to end
original.slice(-2)     // [4, 5] — last 2 elements
original.slice()       // [1,2,3,4,5] — shallow copy of entire array

// concat — merge arrays without mutating
const a = [1, 2];
const b = [3, 4];
const merged = a.concat(b);        // [1, 2, 3, 4] — a and b unchanged
const merged2 = a.concat(b, [5]);  // [1, 2, 3, 4, 5]

// Spread syntax — modern and more readable than concat
const merged3 = [...a, ...b];       // [1, 2, 3, 4]
const withExtra = [...a, 99, ...b]; // [1, 2, 99, 3, 4]
slice vs splice One letter different, completely different behavior. slice is non-mutating and safe. splice mutates the original. A memory trick: splice changes things.

5. Searching Arrays

const scores = [85, 92, 78, 92, 65];

// indexOf — first index of value, -1 if not found
scores.indexOf(92)      // 1 (first occurrence)
scores.indexOf(99)      // -1 (not found)

// lastIndexOf — searches from the end
scores.lastIndexOf(92)  // 3 (last occurrence)

// includes — returns boolean (use this for existence checks)
scores.includes(78)     // true
scores.includes(100)    // false
// Note: indexOf uses ===, so NaN is never found. includes handles NaN correctly.
[NaN].indexOf(NaN)     // -1  — bug!
[NaN].includes(NaN)   // true — correct

// find — returns first ELEMENT passing the test function
const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Karan" },
];
const user = users.find(u => u.id === 2); // { id: 2, name: "Bob" }
const notFound = users.find(u => u.id === 99); // undefined

// findIndex — returns index of first element passing the test
const idx = users.findIndex(u => u.name === "Karan"); // 2

6. Array Transformation

These methods reshape or reorder the array. Know which ones mutate.

// join — convert array to string
["a", "b", "c"].join("-")     // "a-b-c"
["a", "b", "c"].join("")      // "abc"
["a", "b", "c"].join()        // "a,b,c" — default separator is comma

// reverse — MUTATES the array in place
const arr = [1, 2, 3];
arr.reverse(); // arr is now [3, 2, 1] — the original is changed!

// Non-mutating reverse (copy first)
const reversed = [...arr].reverse(); // safe — original untouched

// sort — MUTATES; default sort is ALPHABETICAL (converts to strings first)
[3, 1, 10].sort()                  // [1, 10, 3] ← "10" < "3" alphabetically!

// Correct numeric sort with a comparator function
[3, 1, 10].sort((a, b) => a - b)  // [1, 3, 10] — ascending
[3, 1, 10].sort((a, b) => b - a)  // [10, 3, 1] — descending

// Sorting strings correctly (locale-aware)
["banana", "apple", "cherry"].sort() // ["apple","banana","cherry"] — fine for ASCII

// flat — flatten nested arrays
[1, [2, [3, [4]]]].flat()       // [1, 2, [3, [4]]]    — depth 1 (default)
[1, [2, [3, [4]]]].flat(2)     // [1, 2, 3, [4]]      — depth 2
[1, [2, [3, [4]]]].flat(Infinity) // [1, 2, 3, 4]         — fully flat

// flatMap — map then flat(1) in one step
["hello world", "foo bar"].flatMap(s => s.split(" "))
// ["hello", "world", "foo", "bar"]
Sort Gotcha — the Most Common Array Interview Trap [3, 1, 10].sort() returns [1, 10, 3] because JS converts numbers to strings and sorts lexicographically ("10" comes before "3"). Always provide a comparator when sorting numbers: .sort((a, b) => a - b).

7. Spread & Array Copying

// Shallow copy with spread — preferred modern syntax
const original = [1, 2, 3];
const copy = [...original];
copy.push(4);
console.log(original); // [1, 2, 3] — unchanged
console.log(copy);     // [1, 2, 3, 4]

// Shallow copy with Array.from()
const copy2 = Array.from(original); // same result

// SHALLOW means nested objects still share a reference
const users = [{ name: "Alice" }, { name: "Bob" }];
const usersCopy = [...users];

usersCopy[0].name = "Alicia";     // modifies the SAME object!
console.log(users[0].name);        // "Alicia" — original is affected

// Deep copy (when you need full independence)
const deepCopy = JSON.parse(JSON.stringify(users)); // works for plain data
// structuredClone(users) — modern and handles more types (Node 17+, modern browsers)
Shallow vs Deep Copy Spread ([...arr]) creates a new array, but objects inside it are still the same references. If you mutate a nested object in the copy, the original changes too. For truly independent copies, use structuredClone() or a library like Lodash's _.cloneDeep().
