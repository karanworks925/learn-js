Objects: Basics

Objects are the backbone of JavaScript — almost everything is one.

📖 Primary Source for this lesson Read this after the lesson to solidify what you learn: javascript.info — Objects. The most thorough free resource on this topic — includes visual diagrams and exercises.

1. Creating Objects

An object is a collection of key-value pairs. Keys are always strings (or Symbols); values can be anything — numbers, strings, functions, other objects, arrays, even undefined.

The most common way to create an object is the object literal syntax using curly braces {}.

// Object literal — the bread and butter of JS development
const user = {
  name: "Karan",
  age: 22,
  isAdmin: false,
  address: {               // nested object
    city: "Mumbai",
    zip: "400001"
  },
  greet() {              // method (function as a value)
    console.log("Hello!");
  }
};

// Empty object — perfectly valid, add properties later
const config = {};

There are two property types:

  • Data properties — hold a value (name, age, address above)
  • Method properties — hold a function (greet above)
Key naming rules Keys can be any valid identifier (name, firstName) or any string including spaces and reserved words ("first name", "class"). Numeric keys are automatically converted to strings.

2. Accessing Properties

There are two ways to read a property's value.

Dot Notation

The clean, preferred syntax. Use it whenever the key is a valid identifier you know at write time.

const user = { name: "Karan", age: 22 };

console.log(user.name);   // "Karan"
console.log(user.age);    // 22
console.log(user.email);  // undefined — missing keys return undefined, no error

Bracket Notation

Required in three situations: when the key is stored in a variable, when the key contains spaces or special characters, or when the key is a reserved word.

const user = { name: "Karan", "first name": "Karan", class: "A" };

// Key with a space — dot notation would fail here
console.log(user["first name"]);  // "Karan"

// Dynamic key stored in a variable — this is the big one
const field = "name";
console.log(user[field]);          // "Karan" — evaluates field first
console.log(user.field);           // undefined — looks for key literally "field"

// Reserved word key
console.log(user["class"]);       // "A"

Optional Chaining: ?.

Safely access nested properties without throwing if an intermediate value is null or undefined.

const user = { address: { city: "Mumbai" } };
const guest = {};

console.log(user.address.city);     // "Mumbai"
console.log(guest.address?.city);   // undefined — no error thrown
// Without ?.: guest.address.city throws TypeError: Cannot read properties of undefined

3. Adding, Updating & Deleting

const user = { name: "Karan", age: 22 };

// Adding a new property
user.email = "karan@example.com";
console.log(user.email);   // "karan@example.com"

// Updating an existing property
user.age = 23;
console.log(user.age);    // 23

// Deleting a property
delete user.email;
console.log(user.email);   // undefined

Checking if a Property Exists

Two approaches — and they behave differently. This is an interview question.

const obj = { age: undefined };  // key exists, but value is undefined

// Approach 1: obj.key !== undefined — UNRELIABLE
console.log(obj.age !== undefined);      // false — thinks key is missing!

// Approach 2: "key" in obj — CORRECT (checks for key existence)
console.log("age" in obj);              // true — key exists
console.log("name" in obj);             // false — key doesn't exist
Interview Gotcha "key" in obj also checks the object's prototype chain (inherited properties). To check only own properties, use Object.hasOwn(obj, "key") (modern) or obj.hasOwnProperty("key").

4. ES6 Object Enhancements

Modern JavaScript introduced cleaner shorthand syntax. You'll see all of these in real codebases constantly.

Property Shorthand

const name = "Karan";
const age = 22;

// Old way — repetitive
const user1 = { name: name, age: age };

// ES6 shorthand — when variable name equals key name
const user2 = { name, age };  // identical to user1

Computed Property Names

const field = "score";
const prefix = "user";

const obj = {
  [field]: 100,               // key is the VALUE of field: "score"
  [`${prefix}_id`]: 42       // key is "user_id"
};
console.log(obj.score);     // 100
console.log(obj.user_id);   // 42

Method Shorthand

// Old way
const obj1 = {
  greet: function() { return "Hello"; }
};

// ES6 method shorthand — cleaner, also affects how super works
const obj2 = {
  greet() { return "Hello"; }
};

5. Iterating Over Objects

You cannot use forEach directly on an object — it's for arrays. Use these static Object methods to convert the object into something iterable first.

const scores = { alice: 95, bob: 88, carol: 72 };

// Object.keys — array of keys
console.log(Object.keys(scores));
// ["alice", "bob", "carol"]

// Object.values — array of values
console.log(Object.values(scores));
// [95, 88, 72]

// Object.entries — array of [key, value] pairs
console.log(Object.entries(scores));
// [["alice", 95], ["bob", 88], ["carol", 72]]

// Combine with for...of and destructuring — very common pattern
for (const [name, score] of Object.entries(scores)) {
  console.log(`${name}: ${score}`);
}
// alice: 95
// bob: 88
// carol: 72
Avoid for...in for plain objects for (const key in obj) also iterates over inherited prototype properties, which is almost never what you want. Stick to Object.keys(), Object.values(), and Object.entries() for predictable behavior.

6. Copying & Merging Objects

This is subtle territory. The most important concept: all standard copy methods create shallow copies — nested objects are still shared by reference.

Shallow Copy

const original = { name: "Karan", scores: [90, 85] };

// Spread — the modern approach
const copy1 = { ...original };

// Object.assign — older, same result
const copy2 = Object.assign({}, original);

copy1.name = "Someone";       // ✅ doesn't affect original (primitive)
copy1.scores.push(70);       // ❌ MUTATES original.scores too! (shared reference)
console.log(original.scores); // [90, 85, 70] — surprised?

Deep Copy

const original = { name: "Karan", scores: [90, 85] };

// structuredClone — modern, handles dates, maps, sets, cycles
const deepCopy = structuredClone(original);
deepCopy.scores.push(70);
console.log(original.scores);  // [90, 85] — untouched

// JSON round-trip — old approach, has limitations
const deepCopy2 = JSON.parse(JSON.stringify(original));
// ❌ Loses: functions, undefined values, Date objects (→ strings), Maps, Sets, circular refs

Merging Objects

const defaults = { theme: "light", lang: "en", fontSize: 16 };
const userPrefs = { theme: "dark", fontSize: 18 };

// Later spread wins — userPrefs overrides conflicting keys in defaults
const config = { ...defaults, ...userPrefs };
console.log(config);
// { theme: "dark", lang: "en", fontSize: 18 }

7. this in Object Methods

When a function is stored as a property (a "method") and called on the object, the keyword this inside it refers to the object the method was called on.

const user = {
  name: "Karan",
  age: 22,
  describe() {
    // 'this' is the object to the left of the dot at call time
    return `${this.name} is ${this.age} years old`;
  }
};

console.log(user.describe()); // "Karan is 22 years old"
