Higher-Order Functions

map, filter, reduce — the functional trio that every JS developer must master.

Primary Source for this lesson Read this after the lesson to solidify what you learn: javascript.info — Array Methods. Pay special attention to the reduce examples — they go much deeper than what most tutorials cover.

1. What is a Higher-Order Function?

A higher-order function is a function that does at least one of these two things:

  • Takes a function as an argument (a callback)
  • Returns a function as its result

This is only possible because JavaScript treats functions as first-class values — they can be stored in variables, passed around, and returned just like numbers or strings.

// A simple higher-order function that takes a function as an argument
function doTwice(fn) {
  fn();
  fn();
}

doTwice(() => console.log("hello"));
// hello
// hello

// A higher-order function that returns a function
function multiplier(factor) {
  return (num) => num * factor;
}

const double = multiplier(2);
const triple = multiplier(3);

double(5); // 10
triple(4); // 12

The built-in array methods map, filter, reduce, forEach, find, some, and every are all higher-order functions. They take a callback and apply it across the array. This is one of the most important patterns in all of JavaScript.

2. forEach — Iterate with Side Effects

forEach calls your callback once for each element. It is the functional replacement for a for loop when you want to do something with each element but don't need a new array back.

const fruits = ["apple", "banana", "cherry"];

fruits.forEach((item, index, array) => {
  console.log(`${index}: ${item}`);
});
// 0: apple
// 1: banana
// 2: cherry

// forEach always returns undefined
const result = fruits.forEach(x => x);
console.log(result); // undefined
forEach is not chainable Because forEach always returns undefined, you cannot chain another method after it. If you need to chain, use map instead.

3. map — Transform Every Element

map creates a new array by calling your callback on each element and collecting the return values. The original array is never modified. The new array always has the same length as the original.

const numbers = [1, 2, 3, 4];

// Double every number
const doubled = numbers.map(n => n * 2);
console.log(doubled);  // [2, 4, 6, 8]
console.log(numbers);  // [1, 2, 3, 4] — original unchanged

// Extract a property from each object
const users = [
  { name: "Alice", age: 28 },
  { name: "Bob", age: 34 },
  { name: "Carol", age: 22 },
];

const names = users.map(user => user.name);
console.log(names); // ["Alice", "Bob", "Carol"]

// Format data for display
const formatted = users.map(user => ({
  label: `${user.name} (${user.age})`,
  value: user.name.toLowerCase(),
}));
// [{ label: "Alice (28)", value: "alice" }, ...]
When to use map Reach for map whenever you need a transformed version of an array. "I want the same number of items, just different." That's map.

4. filter — Keep Elements That Pass a Test

filter creates a new array containing only the elements for which your callback returned a truthy value. The result may be shorter (or even empty) than the original.

const numbers = [1, 2, 3, 4, 5, 6];

// Keep only even numbers
const evens = numbers.filter(n => n % 2 === 0);
console.log(evens); // [2, 4, 6]

// Filter objects by condition
const users = [
  { name: "Alice", active: true },
  { name: "Bob", active: false },
  { name: "Carol", active: true },
];

const activeUsers = users.filter(u => u.active);
// [{ name: "Alice", active: true }, { name: "Carol", active: true }]

// Remove nullish values from an array
const mixed = [1, null, 2, undefined, 3, "", 4];
const clean = mixed.filter(Boolean); // passes each value through Boolean()
console.log(clean); // [1, 2, 3, 4]
filter(Boolean) Passing the Boolean constructor directly as a callback is a common idiom. It's equivalent to filter(x => Boolean(x)) — removes all falsy values.

5. reduce — Accumulate to a Single Value

reduce is the most powerful and flexible of the trio. It processes the array element by element, threading an accumulator through each step, and returns a single final value. That value can be a number, string, object, or even another array.

// Signature: arr.reduce((accumulator, currentItem, index, array) => ..., initialValue)

const nums = [1, 2, 3, 4];

// Sum all numbers
const sum = nums.reduce((acc, n) => acc + n, 0);
console.log(sum); // 10

// Find the maximum
const max = nums.reduce((acc, n) => (n > acc ? n : acc), -Infinity);
console.log(max); // 4

// Count occurrences of each value
const letters = ["a", "b", "a", "c", "b", "a"];
const counts = letters.reduce((acc, letter) => {
  acc[letter] = (acc[letter] || 0) + 1;
  return acc;
}, {});
console.log(counts); // { a: 3, b: 2, c: 1 }

// Group objects by a property
const people = [
  { name: "Alice", dept: "eng" },
  { name: "Bob", dept: "design" },
  { name: "Carol", dept: "eng" },
];
const byDept = people.reduce((acc, person) => {
  const key = person.dept;
  acc[key] = acc[key] || [];
  acc[key].push(person);
  return acc;
}, {});
// { eng: [Alice, Carol], design: [Bob] }
Always provide the initial value If you omit the initial value, reduce uses the first element as the accumulator and starts iterating from index 1. This breaks when the array has one element and blows up on an empty array. Always pass the second argument.

6. find and findIndex

find returns the first element that satisfies the callback. If nothing matches, it returns undefined. findIndex does the same but returns the index instead (-1 if not found).

const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Carol" },
];

const found = users.find(u => u.id === 2);
console.log(found); // { id: 2, name: "Bob" }

const idx = users.findIndex(u => u.id === 2);
console.log(idx); // 1

// vs indexOf — indexOf uses === (strict equality on the value itself)
const nums = [10, 20, 30];
nums.indexOf(20);          // 1 — works for primitives
nums.findIndex(n => n > 15); // 1 — find lets you use any condition

7. some and every

These two return a boolean. Both short-circuit — they stop iterating as soon as they have a definitive answer.

const scores = [72, 88, 45, 91, 60];

// some: true if AT LEAST ONE passes the test
const anyFailing = scores.some(s => s < 50);
console.log(anyFailing); // true (45 fails)

// every: true only if ALL pass the test
const allPassing = scores.every(s => s >= 50);
console.log(allPassing); // false (45 does not pass)

// Short-circuit: every stops the moment it finds a false
const allPositive = [1, 2, -1, 3].every(n => {
  console.log("checking", n);
  return n > 0;
});
// checking 1
// checking 2
// checking -1 ← stops here, never checks 3

8. Chaining

Because map and filter return new arrays, you can chain them together to build readable data pipelines. Think of each step as a transformation stage.

const orders = [
  { product: "Laptop", price: 1200, shipped: true },
  { product: "Mouse", price: 25, shipped: false },
  { product: "Monitor", price: 400, shipped: true },
  { product: "Keyboard", price: 80, shipped: true },
];

// Total revenue from shipped orders over $100
const revenue = orders
  .filter(o => o.shipped)          // keep shipped
  .filter(o => o.price > 100)     // keep expensive
  .map(o => o.price)               // extract prices
  .reduce((sum, p) => sum + p, 0); // sum them

console.log(revenue); // 1600 (Laptop + Monitor)
Performance tip Put filter before map when chaining — reducing the array length first means map does less work. This matters on large datasets.
