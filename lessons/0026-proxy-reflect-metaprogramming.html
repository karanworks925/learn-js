Proxy, Reflect & Meta-programming

Intercept and redefine fundamental JavaScript operations — this is how frameworks are built.

Primary Source for this lesson Read this after the lesson to solidify what you learn: javascript.info — Proxy and Reflect. This is the definitive deep-dive into JavaScript's meta-programming APIs.

1. What is Meta-programming?

Meta-programming is writing code that operates on other code — inspecting it, modifying its behavior, or intercepting how it works at runtime. In JavaScript, the primary tools for this are:

  • Proxy — wrap any object and intercept fundamental operations on it
  • Reflect — a set of methods that mirror Proxy traps, for use inside those traps
  • Symbols — unique keys that let you add behavior hooks to objects (like Symbol.iterator)
  • Property descriptors — control how individual properties behave

This is not niche knowledge. Vue 3's entire reactivity system, many ORMs, validation libraries, and testing tools are built directly on these APIs. Understanding them puts you in the same mental space as framework authors.

Why This Matters for Interviews Senior-level interviews frequently ask "how would you implement X" — and the answer is often Proxy. Questions like "how does Vue's reactivity work?" or "how would you build a validation layer?" both lead here.

2. Property Descriptors

Every property on a JavaScript object has a hidden descriptor — an object that defines how that property behaves. You've been working with properties your whole JS career without seeing these, but they've always been there.

const user = { name: "Karan" };

// Look at the descriptor for 'name':
Object.getOwnPropertyDescriptor(user, "name");
// {
//   value: "Karan",
//   writable: true,
//   enumerable: true,
//   configurable: true
// }

// Define a non-writable property:
Object.defineProperty(user, "id", {
  value: 42,
  writable: false,
  enumerable: true,
  configurable: false
});

user.id = 99;         // silently fails (throws in strict mode)
console.log(user.id); // 42 — unchanged
Descriptor KeyWhat It Controls
valueThe property's value
writableWhether assignment (=) can change the value
enumerableWhether it appears in for...in and Object.keys()
configurableWhether the descriptor itself can be changed or the property deleted
Getter / Setter Descriptors Instead of value + writable, you can use get + set functions. This is how computed properties work in classes. The two forms are mutually exclusive.

3. Object.freeze, Object.seal, Object.preventExtensions

These three methods apply constraints to an entire object at once, using property descriptors under the hood.

MethodAdd new props?Delete props?Write existing?
Object.freeze(obj)NoNoNo
Object.seal(obj)NoNoYes
Object.preventExtensions(obj)NoYesYes
 
const config = Object.freeze({
  apiUrl: "https://api.example.com",
  timeout: 5000
});

config.timeout = 9999;       // silently ignored
config.newProp = "hello";    // silently ignored
delete config.apiUrl;        // silently ignored

console.log(config.timeout); // 5000 — frozen
Freeze is Shallow Object.freeze() only freezes the top level. Nested objects are NOT frozen — they can still be mutated. For deep freezing, you need a recursive implementation. 
function deepFreeze(obj) {
  Object.getOwnPropertyNames(obj).forEach(name => {
    const val = obj[name];
    if (typeof val === "object" && val !== null) {
      deepFreeze(val);
    }
  });
  return Object.freeze(obj);
}

4. The Proxy Object

new Proxy(target, handler) creates a wrapper around target. The handler is an object containing traps — functions that intercept fundamental operations before they reach the target.

const target = { name: "Karan", age: 25 };

const handler = {
  get(target, prop) {
    console.log(`Reading: ${prop}`);
    return Reflect.get(target, prop);
  },
  set(target, prop, value) {
    console.log(`Writing: ${prop} = ${value}`);
    return Reflect.set(target, prop, value);
  }
};

const proxy = new Proxy(target, handler);

proxy.name;       // logs "Reading: name", returns "Karan"
proxy.age = 26;  // logs "Writing: age = 26"

Common Proxy Traps

TrapInterceptsSignature

Validation Proxy — Practical Example

function createUser(data) {
  return new Proxy(data, {
    set(target, prop, value) {
      if (prop === "age") {
        if (typeof value !== "number" || value < 0 || value > 150) {
          throw new TypeError(`Invalid age: ${value}`);
        }
      }
      if (prop === "name" && typeof value !== "string") {
        throw new TypeError("Name must be a string");
      }
      return Reflect.set(target, prop, value);
    }
  });
}

const user = createUser({ name: "Karan", age: 25 });
user.age = 26;        // fine
user.age = "old";     // TypeError: Invalid age: old
user.age = -5;        // TypeError: Invalid age: -5

5. The Reflect API

The Reflect object is a collection of static methods that mirror every Proxy trap, one-to-one. Its purpose is to give you a clean, reliable way to perform the default operation inside a trap.

// Instead of doing this inside a get trap:
get(target, prop) {
  return target[prop];  // risky — loses receiver context
}

// Do this:
get(target, prop, receiver) {
  return Reflect.get(target, prop, receiver);  // correct
}
Why Reflect over Direct Access? Reflect methods properly forward the receiver (the this value for getters/setters), return boolean success values instead of throwing, and maintain consistency with the internal JavaScript specification operations. Always use Reflect inside Proxy traps.
Reflect MethodEquivalent To
Reflect.get(target, prop, receiver)target[prop]
Reflect.set(target, prop, value, receiver)target[prop] = value
Reflect.has(target, prop)prop in target
Reflect.deleteProperty(target, prop)delete target[prop]
Reflect.ownKeys(target)Object.getOwnPropertyNames() + Symbols
Reflect.apply(target, thisArg, args)target.apply(thisArg, args)

6. Real-world Proxy Use Cases

Reactive Data (How Vue 3 Works)

function reactive(obj) {
  const subscribers = new Map();

  return new Proxy(obj, {
    get(target, prop, receiver) {
      // Track: something is reading this property
      return Reflect.get(target, prop, receiver);
    },
    set(target, prop, value, receiver) {
      const result = Reflect.set(target, prop, value, receiver);
      // Notify: trigger UI re-render for this property
      console.log(`[reactive] ${prop} changed to ${value} — re-rendering`);
      return result;
    }
  });
}

const state = reactive({ count: 0 });
state.count++;  // logs: [reactive] count changed to 1 — re-rendering

Default Values for Missing Properties

const withDefaults = new Proxy({}, {
  get(target, prop) {
    return prop in target ? target[prop] : "(not set)";
  }
});

withDefaults.name = "Karan";
console.log(withDefaults.name);   // "Karan"
console.log(withDefaults.email);  // "(not set)" — no error!

Access Logging / Auditing

function auditProxy(obj, label) {
  return new Proxy(obj, {
    get(target, prop, receiver) {
      console.log(`[AUDIT] ${label}.${prop} read`);
      return Reflect.get(target, prop, receiver);
    },
    set(target, prop, value, receiver) {
      console.log(`[AUDIT] ${label}.${prop} set to ${JSON.stringify(value)}`);
      return Reflect.set(target, prop, value, receiver);
    }
  });
}

const sensitiveData = auditProxy({ token: "abc123" }, "credentials");
sensitiveData.token;  // [AUDIT] credentials.token read

7. Revocable Proxies

A normal Proxy lives as long as the target object. A revocable Proxy can be explicitly disabled — after which any operation on it throws a TypeError.

const { proxy, revoke } = Proxy.revocable(
  { secret: "classified" },
  {
    get(target, prop) {
      return Reflect.get(target, prop);
    }
  }
);

console.log(proxy.secret); // "classified"

revoke(); // kill the proxy

console.log(proxy.secret); // TypeError: Cannot perform 'get' on a proxy that has been revoked
Use Case: Temporary Access Revocable proxies are perfect for granting temporary access to an API or resource. When the session ends, call revoke() — the caller's reference is now permanently disabled even if they held onto it.
