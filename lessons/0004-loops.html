Loops

Repetition is the engine of programming — know all 5 ways JS does it.

Primary Source for this lesson Read this after the lesson: javascript.info — while and for loops. It covers all loop types with clear diagrams and exercises.

1. The Classic for Loop

The most explicit loop — you control the start, the stop condition, and the update step. Best when you know exactly how many times to loop, or when you need the index.

// Anatomy: for (initialization; condition; update)
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4
}

// Iterating an array by index
const fruits = ["apple", "banana", "cherry"];
for (let i = 0; i < fruits.length; i++) {
  console.log(i, fruits[i]); // 0 "apple", 1 "banana", 2 "cherry"
}

// Counting down
for (let i = 10; i >= 0; i--) {
  console.log(i); // 10, 9, 8, ... 0
}

// Nested loops — O(n²) complexity, use with awareness
for (let row = 0; row < 3; row++) {
  for (let col = 0; col < 3; col++) {
    console.log(`[${row}][${col}]`);
  }
} // 9 iterations total — grows quadratically
Common Bug Off-by-one errors: i < arr.length is correct for zero-indexed arrays. i <= arr.length will access arr[arr.length], which is undefined.

2. while Loop

Use while when you do not know in advance how many iterations are needed — you loop as long as a condition holds true.

// Basic while loop
let i = 0;
while (i < 5) {
  console.log(i); // 0, 1, 2, 3, 4
  i++;             // must update or you get an infinite loop
}

// Practical: retry logic — keep trying until success or max attempts
let attempts = 0;
let connected = false;

while (!connected && attempts < 3) {
  connected = tryConnect();  // some function that may succeed or fail
  attempts++;
}

// Reading until a sentinel value
let input = getNextLine();
while (input !== "quit") {
  processLine(input);
  input = getNextLine();
}
Infinite Loop Danger Every while loop must have a path to make its condition false. If the condition never becomes false and there is no break, the loop runs forever and freezes the browser/process. Always double-check that your exit condition is reachable.

3. do...while Loop

Like while, but the body runs before the condition is checked — so it always executes at least once.

// do...while — always executes once, then checks
let count = 10;
do {
  console.log(count); // runs ONCE even though count >= 5 already
  count++;
} while (count < 5); // logs just 10, then stops

// Classic use case: prompt until valid input
let answer;
do {
  answer = prompt("Enter a number between 1 and 10:");
} while (answer < 1 || answer > 10);
console.log("You entered:", answer);
When to Use do...while When the action must happen at least once before you can know whether to continue — like showing a form before validating, or making a first network request before checking whether to retry.

4. for...of Loop — Modern Default for Arrays

for...of iterates over any iterable — arrays, strings, Maps, Sets, NodeLists — giving you the values directly. Prefer this over classic for loops when you do not need the index.

// Iterating array values
const scores = [88, 92, 75, 96];
for (const score of scores) {
  console.log(score); // 88, 92, 75, 96
}

// Getting both index and value with .entries()
for (const [i, score] of scores.entries()) {
  console.log(`#${i + 1}: ${score}`); // "#1: 88", "#2: 92", ...
}

// Works on strings — iterates characters
for (const ch of "hello") {
  console.log(ch); // 'h', 'e', 'l', 'l', 'o'
}

// Works on Map — gives [key, value] pairs
const userRoles = new Map([
  ["Karan", "admin"],
  ["Priya", "editor"],
]);
for (const [name, role] of userRoles) {
  console.log(`${name} is ${role}`);
}

// Works on Set — unique values
const unique = new Set([1, 2, 2, 3]);
for (const val of unique) {
  console.log(val); // 1, 2, 3 — duplicates are skipped
}
Default Choice In modern JS, reach for for...of first when iterating arrays. Use for (classic) only when you specifically need the numeric index for reasons beyond reading the value.

5. for...in Loop — For Objects, Use Carefully

for...in iterates over the enumerable property keys of an object. It is designed for plain objects — not arrays.

// for...in on an object — gets keys as strings
const person = { name: "Karan", age: 22, city: "Delhi" };

for (const key in person) {
  console.log(key, person[key]);
  // "name" "Karan"
  // "age"  22
  // "city" "Delhi"
}

// Safer: use Object.keys(), Object.values(), Object.entries()
for (const [key, val] of Object.entries(person)) {
  console.log(`${key}: ${val}`);
}
// WHY for...in is dangerous on arrays
const arr = ["a", "b", "c"];
arr.customProp = "oops"; // someone added a property to the array object

for (const key in arr) {
  console.log(key); // "0", "1", "2", "customProp" ← unexpected!
}
// Keys are STRINGS, and inherited or custom properties show up too

// Also inherits from the prototype chain:
// hasOwnProperty guard (old approach)
for (const key in person) {
  if (person.hasOwnProperty(key)) { // skip inherited properties
    console.log(key);
  }
}
Never use for...in on arrays It gives you string keys ("0", "1", "2"), iterates any extra properties added to the array, and may include inherited prototype properties. Use for...of or forEach for arrays. Use Object.keys()/Object.entries() for objects.

6. break and continue

// break — exit the loop immediately
const users = ["alice", "bob", "karan", "diana"];
let found = null;

for (const user of users) {
  if (user === "karan") {
    found = user;
    break; // stop looping — no need to check the rest
  }
}
console.log(found); // "karan"

// continue — skip to the next iteration
const numbers = [1, 2, 3, 4, 5, 6];
for (const n of numbers) {
  if (n % 2 === 0) continue; // skip even numbers
  console.log(n); // 1, 3, 5
}
// Labeled break — exit an OUTER loop from an inner loop
// (Rarely needed — use with a comment explaining why)
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) {
      break outer; // exits both loops at once
    }
    console.log(i, j);
  }
}
// Logs: 0 0, 0 1, 0 2, 1 0 — then stops entirely
