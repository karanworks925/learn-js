# JavaScript Resources

## Knowledge

- [MDN Web Docs — JavaScript](https://developer.mozilla.org/en-US/docs/Web/JavaScript)
  The primary reference for all JS. Use for: syntax, built-in methods, browser APIs, specs. Highest-trust source available.

- [JavaScript.info](https://javascript.info)
  Comprehensive, modern JS tutorial by Ilya Kantor. Exceptional depth and clarity. Use for: understanding *why* JS behaves the way it does, from basics to advanced internals.

- [You Don't Know JS (Yet) — Kyle Simpson (free on GitHub)](https://github.com/getify/You-Dont-Know-JS)
  Deep-dive into JS mechanics (scope, closures, `this`, prototypes, async). Use for: interview-level understanding, filling in gaps left by surface-level tutorials.

- [Eloquent JavaScript — Marijn Haverbeke (free online)](https://eloquentjavascript.net)
  Thorough textbook with exercises. Use for: structured learning, problem-solving skills, algorithms in JS context.

- [ECMAScript Specification — TC39](https://tc39.es/ecma262/)
  The official language spec. Use for: definitively settling behavior questions. Not for casual reading.

- [What the heck is the event loop anyway? — Philip Roberts (JSConf EU, YouTube)](https://www.youtube.com/watch?v=8aGhZQkoFbQ)
  The clearest explanation of the JS event loop that exists. Use for: event loop, call stack, microtask/macrotask queue.

## Wisdom (Communities)

- [r/learnjavascript](https://reddit.com/r/learnjavascript)
  Welcoming, moderated community for learners. Use for: getting unstuck, asking "why does this work?" questions.

- [Stack Overflow — javascript tag](https://stackoverflow.com/questions/tagged/javascript)
  Use for: debugging specific issues, seeing how common problems are solved in the wild.

- [Discord: Reactiflux / Programming Hangout](https://discord.gg/reactiflux)
  Large, active JS/web dev community. Use for: real-time help, code review from working developers.
