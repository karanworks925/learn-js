# Mission: JavaScript — Complete Mastery for a Dev Job

## Why
Karan has some existing JS exposure and wants to land a developer job. That means understanding JavaScript deeply enough to write production-quality code AND to explain every concept clearly in a technical interview. The goal is not just to make things work — it's to know *why* they work.

## Success looks like
- Pass technical phone screens and coding interviews involving JavaScript
- Write clean, idiomatic ES6+ JavaScript without looking things up
- Explain tricky JS concepts (closures, the event loop, `this`, prototypes) confidently under pressure
- Build and ship a real project in JavaScript that can go in a portfolio
- Debug JavaScript problems by understanding root causes, not guessing

## Constraints
- Has some prior JS experience — do not waste time on things already internalized
- Needs interview-readiness: every lesson should build toward something that can come up in a job interview
- Lessons should be completable in short sessions (20–30 min each)

## Out of scope
- Specific frameworks (React, Vue, etc.) — pure JS mastery first
- TypeScript — after core JS is solid
- Backend/Node.js in depth — revisit after foundations are complete
